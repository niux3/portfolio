Associer plusieurs valeurs à une même clé dans un dictionnaire

Vous cherchez à créer un dictionnaire capable d’associer plusieurs valeurs à une même clé, autrement dit un « multidictionnaire ». Un dictionnaire est une structure de données qui associe chaque clé à une seule valeur. Si vous souhaitez associer une clé à plusieurs valeurs, vous devez stocker ces valeurs multiples dans un autre conteneur, comme une liste ou un ensemble set. Par exemple, vous pouvez créer des dictionnaires comme ceci :

d = {
    'a' : [1, 2, 3],
    'b' : [4, 5]
}
e = {
    'a' : {1, 2, 3},
    'b' : {4, 5}
}

Le choix entre utiliser des listes ou des ensembles set dépend de l’usage que vous souhaitez en faire. Utilisez une liste si vous voulez préserver l’ordre d’insertion des éléments. Lorsque vous utilisez un set pour stocker plusieurs valeurs associées à une même clé dans un dictionnaire, il est important de garder à l’esprit que ce type de conteneur ne conserve pas l’ordre des éléments et ne permet pas les doublons. Ainsi, si vous ajoutez plusieurs fois la même valeur, elle ne sera stockée qu’une seule fois. Le comportement du set est un bon choix lorsque vous avez besoin de garantir l’unicité des valeurs, comme dans le cas de catégories sans répétition, de droits d’accès ou des labels uniques.

Pour construire facilement ce type de dictionnaires (où une clé est associée à plusieurs valeurs), vous pouvez utiliser la classe defaultdict du module collections. L’un des avantages de defaultdict, c’est qu’il initialise automatiquement la première valeur pour chaque nouvelle clé, ce qui vous permet de vous concentrer uniquement sur l’ajout des éléments. Par exemple :

from collections import defaultdict


d = defaultdict(list)
d['a'].append(1)
d['a'].append(2)
d['a'].append(2)
d['b'].append(4)

print(d)

d = defaultdict(set)
d['a'].add(1)
d['a'].add(2)
d['a'].add(2)
d['b'].add(4)

print(d)

Attention : defaultdict crée automatiquement des entrées dans le dictionnaire pour toute clé accédée, même si cette clé n’existait pas auparavant. Si vous ne souhaitez pas ce comportement, vous pouvez plutôt utiliser la méthode setdefault() sur un dictionnaire standard (dict). Par exemple :

d = {} # A regular dictionary

d.setdefault('a', []).append(1)
d.setdefault('a', []).append(2)
d.setdefault('b', []).append(4)

Cependant, de nombreux développeurs trouvent que l’utilisation de setdefault() est un peu contre-intuitive. Sans parler du fait qu’elle crée à chaque appel une nouvelle instance de la valeur initiale (par exemple, une liste vide [] dans l’exemple).

En principe, la construction d’un dictionnaire à valeurs multiples est simple. Cependant, l’initialisation de la première valeur peut devenir fastidieuse si vous tentez de la gérer manuellement. Par exemple, vous pourriez vous retrouver avec un code ressemblant à ceci :

d = {}
for key, value in pairs:
    if key not in d:
        d[key] = []
    d[key].append(value)

L’utilisation d’un defaultdict permet simplement d’obtenir un code beaucoup plus clair et lisible :

d = defaultdict(list)
for key, value in pairs:
    d[key].append(value)
