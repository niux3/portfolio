« Flask, c’est amusant. » Cette déclaration audacieuse est l'une des premières choses que vous verrez en consultant la documentation officielle de Flask et, au fil de ce tuto, vous comprendrez pourquoi tant de développeurs Python sont du même avis.

Qu’est-ce que Flask ?

Flask est un framework Web léger écrit en Python. Il a commencé comme une blague du 1er avril, avant de devenir un outsider très populaire dans le monde des frameworks web en Python. Aujourd’hui, c’est l’un des frameworks Python les plus utilisés par les start-ups, et il est de plus en plus reconnu comme l’outil idéal pour développer des solutions simples et rapides dans la plupart des entreprises. Au cœur de Flask, on trouve un ensemble de bibliothèques puissantes permettant de gérer les tâches les plus courantes du développement web, telles que :

  • Le routage d’URL, qui permet de lier facilement des URLs à votre code
  • Le rendu de templates avec Jinja2, l’un des moteurs de templates Python les plus puissants
  • La gestion des sessions et la sécurisation des cookies
  • L’analyse des requêtes HTTP et une gestion flexible des réponses
  • Un débogueur interactif accessible via le navigateur
  • Une gestion de configuration d’application simple, souple et facile à utiliser

Ce tuto vous apprendra à utiliser ces outils à travers des exemples pratiques et concrets. Nous aborderons également des bibliothèques tierces couramment utilisées pour les fonctionnalités que Flask ne propose pas en natif, comme l’accès aux bases de données ou la validation de formulaires. À la fin de ce tuto, vous serez prêt à vous lancer dans votre prochain grand projet avec Flask.

Une grande liberté implique une grande responsabilité

Comme le dit la documentation, Flask, c’est amusant, mais cela peut aussi être un défi, en particulier lorsque vous développez une application de grande envergure. Contrairement à d’autres frameworks web populaires en Python, comme Django, Flask n’impose pas de structure particulière pour organiser vos modules ou votre code. Si vous avez déjà utilisé d’autres frameworks web, vous pourriez être surpris de constater que développer avec Flask ressemble davantage à écrire du Python pur qu’à remplir un canevas imposé par le framework.

Ce tuto vous apprendra à utiliser Flask pour écrire des applications claires et expressives. Au fil de votre progression, non seulement vous deviendrez un développeur Flask expérimenté, mais vous renforcerez aussi vos compétences en développement Python

Mise en place d’un environnement de développement

Flask est écrit en Python, donc avant de commencer à créer des applications Flask, nous devons nous assurer que Python est installé. La plupart des distributions Linux et les versions récentes de macOS sont livrées avec Python préinstallé. Les exemples de ce tuto nécessitent Python 3.x. Des instructions pour installer Python sont disponibles à l’adresse suivante : http://www.python.org.

Si c’est la première fois que vous utilisez Python, de nombreuses ressources gratuites et de qualité sont disponibles sur le web. Je vous recommande Apprendre à programmer avec Python 3, de Gérard Swinnen, accessible gratuitement.

Vous pouvez vérifier que Python est installé et que vous disposez de la bonne version en lançant l’interpréteur interactif Python depuis un terminal :

$python3
Python 3.9.2 (default, Mar 20 2025, 02:07:39) 
[GCC 10.2.1 20210110] on linux
Type "help", "copyright", "credits" or "license" for more information.
>>>

À l’invite de commande (>>>), tapez exit() puis appuyez sur Entrée pour quitter l’interpréteur.

Installation de packages Python

Maintenant que vous vous êtes assuré que Python est correctement installé, nous allons installer quelques packages Python populaires qui seront utilisés tout au long de ce tuto.

Nous allons d’abord installer ces packages à l’échelle du système, mais une fois cela fait, nous travaillerons exclusivement dans des environnements virtuels.

Installation de pip

L’installateur de packages Python de référence est pip. Nous l’utiliserons tout au long de ce tuto pour installer Flask et d’autres bibliothèques tierces.

Si vous avez déjà setuptools installé, vous pouvez installer pip en lançant simplement la commande suivante :

$sudo easy_install pip

Après avoir terminé l’installation, vérifiez que pip est correctement installé :

$pip --version
pip 22.1 from /home/$user/.local/lib/python3.9/site-packages/pip (python 3.9)

Les numéros de version sont susceptibles d’évoluer, donc pour un guide définitif, veuillez consulter les instructions officielles disponibles à l’adresse suivante : http://www.pip-installer.org/en/latest/installing.html.

Installation de virtualenv

Une fois pip installé, nous pouvons passer à l’installation de l’outil le plus important dans la boîte à outils de tout développeur Python : virtualenv. Virtualenv facilite la création d’environnements Python isolés, chacun avec ses propres copies des packages système et tiers.

Pourquoi utiliser virtualenv ?

Virtualenv résout plusieurs problèmes liés à la gestion des packages. Imaginez que vous ayez une vieille application construite avec une toute première version de Flask, et que vous souhaitiez créer un nouveau projet avec la version la plus récente de Flask. Si Flask est installé à l’échelle du système, vous seriez obligé soit de mettre à jour votre ancien projet, soit d’écrire votre nouveau projet en utilisant l’ancienne version de Flask. Si les deux projets utilisaient virtualenv, chacun pourrait fonctionner avec sa propre version de Flask, sans conflits ni problèmes. Virtualenv facilite le contrôle des versions des packages tiers utilisés par votre projet.

Autre point important : installer des packages à l’échelle du système nécessite généralement des privilèges élevés (par exemple, sudo pip install foo). En utilisant virtualenv, vous pouvez créer des environnements Python et installer des packages en tant qu’utilisateur normal. Cela est particulièrement utile si vous déployez sur un hébergement mutualisé ou si vous n’avez pas les droits d’administrateur.

Installation de virtualenv avec pip

Nous allons utiliser pip pour installer virtualenv ; comme il s’agit d’un package Python standard, il peut être installé comme n’importe quel autre package Python. Pour vous assurer que virtualenv est installé à l’échelle du système, exécutez la commande suivante (qui nécessite des privilèges élevés) :

$sudo pip install virtualenv
$virtualenv --version
virtualenv 20.14.1 from /home/$user/.local/lib/python3.9/site-packages/virtualenv/__init__.py

Les numéros de version sont susceptibles d’évoluer, donc pour un guide définitif, veuillez consulter les instructions officielles sur : http://virtualenv.org.

Créer votre première application Flask

Maintenant que nous avons installé les outils nécessaires, nous sommes prêts à créer notre première application Flask. Pour commencer, créez un répertoire à un emplacement pratique qui contiendra tous vos projets Python. Dans le terminal ou l’invite de commande, placez-vous dans ce répertoire ; pour ma part, c’est /home/$user/projects, ou simplement ~/projects sur les systèmes Unix.

$mkdir projects
$cd projects

Nous allons maintenant créer un environnement virtuel (virtualenv). Les commandes ci-dessous vont créer un nouveau répertoire nommé hello_flask à l’intérieur de votre dossier de projets, contenant un environnement Python complet et isolé.

$virtualenv venv

Vous devriez avoir un nouveau dossier du nom de venv à la racine de ~ /projects. Maintenant, il va falloir activer cet environnement virtuel. 

$source venv/bin/activate

Vous devriez constater que votre prompt dans votre terminal a changé. En effet, avant les droits, le nom du dossier venv est entre parenthèses.

note: Dans un environnement Microsoft, ça n'est pas le cas ! 

(venv) user@host:~/projects$

Lorsque vous activez un virtualenv, votre variable d’environnement PATH est temporairement modifiée afin de garantir que les packages que vous installez ou utilisez restent confinés à cet environnement virtuel.

Installer Flask dans votre virtualenv

Maintenant que nous avons vérifié que notre environnement virtuel est correctement configuré, nous pouvons installer Flask. Lorsque vous êtes dans un virtualenv, vous ne devez jamais installer de packages avec des privilèges administrateur. Si vous obtenez une erreur de permission lors de l’installation de Flask, vérifiez bien que votre virtualenv est activé correctement.

(venv)$ pip install Flask

Vous verrez défiler du texte pendant que pip télécharge le package Flask ainsi que ses dépendances avant de les installer dans votre environnement virtuel. Flask dépend de quelques bibliothèques tierces supplémentaires, que pip téléchargera et installera automatiquement pour vous. Vérifions maintenant que tout a bien été installé :

(venv) user@host:~/projects$ python
Python 3.9.2 (default, Mar 20 2025, 02:07:39) 
[GCC 10.2.1 20210110] on linux

Félicitations ! Vous avez installé Flask, et nous sommes maintenant prêts à commencer à coder.

Hello, Flask!

Créez un nouveau fichier nommé app.py dans le répertoire de votre virtualenv venv. À l’aide de votre éditeur de texte ou IDE préféré, saisissez le code suivant :

from flask import Flask


app = Flask(__name__)

@app.route("/")
def hello():
    return "Hello, Flask!"

if __name__ == '__main__':
    app.run(debug=True)

Enregistrez le fichier, puis exécutez app.py en le lançant depuis la ligne de commande. Vous devez vous assurer que votre environnement virtuel venv est bien activé :

(venv) user@host:~/projects$ python app.py 
 * Serving Flask app 'app'
 * Debug mode: on
WARNING: This is a development server. Do not use it in a production deployment. Use a production WSGI server instead.
 * Running on http://127.0.0.1:5000
Press CTRL+C to quit
 * Restarting with stat
 * Debugger is active!
 * Debugger PIN: 110-019-933

Lorsque vous ouvrez votre navigateur préféré et que vous accédez à l’adresse http://127.0.0.1:5000, vous verrez s’afficher une page blanche très simple, sans mise en forme ni image, avec uniquement le message suivant au centre (ou en haut à gauche selon le navigateur) :

Hello, Flask!

Ce rendu minimal est tout à fait normal : Flask renvoie simplement une chaîne de caractères brute sans HTML ni CSS. C’est le comportement de base d’une route Flask renvoyant du texte simple.

Comprendre le code

Nous venons de créer une application Flask très basique. Pour bien comprendre ce qui se passe, analysons ce code ligne par ligne :

from flask import Flask

Notre application commence par importer la classe Flask. Cette classe représente une application WSGI (Web Server Gateway Interface) unique, et c’est l’objet central de tout projet Flask.

Qu’est-ce que WSGI ?

WSGI est le standard officiel pour les serveurs web Python, défini dans la PEP 333. On peut le voir comme un ensemble de règles et de comportements que doivent respecter une application et un serveur pour qu’ils puissent communiquer correctement.

En d’autres termes, WSGI agit comme un intermédiaire :

  • entre ton code Python (ton app Flask),
  • et un serveur web (comme Gunicorn, uWSGI, ou même le serveur de développement de Flask).

Bonne nouvelle : Flask s’occupe de toute cette complexité pour vous. Vous n’avez pas besoin d’implémenter WSGI manuellement. Vous vous concentrez uniquement sur le développement de votre application, et Flask gère l’interface avec les serveurs.

app = Flask(__name__)

Dans cette ligne, nous créons une instance de l'application Flask dans la variable app, en lui passant le nom de notre module (généralement via __name__).

  • app : c’est le nom de la variable dans laquelle on stocke notre application Flask. Ce nom est une convention courante, mais on pourrait l’appeler autrement si on le souhaite.
  • Flask(__name__) : on instancie Flask en lui passant __name__, qui est une variable spéciale contenant le nom du module actuel (par exemple __main__ si vous exécutez le fichier directement).
@app.route("/")
def hello():
    return "Hello, Flask!"

Dans les lignes précédentes, nous indiquons à notre application Flask de diriger toutes les requêtes vers / (l’URL racine) vers cette fonction de vue (index). Une vue est simplement une fonction ou une méthode qui retourne une réponse quelconque. Chaque fois que vous ouvrez un navigateur et accédez à l’URL racine de notre application, Flask appellera cette fonction de vue et enverra la valeur retournée au navigateur. Il y a quelques points à noter concernant ces lignes de code :

  • @app.route est un décorateur Python lié à la variable app définie précédemment. Ce décorateur (app.route) enveloppe la fonction suivante, ici index, afin de diriger les requêtes pour une URL particulière vers une vue spécifique. Le nom index est choisi pour cette fonction car c’est le nom courant pour la première page qu’un serveur web utilise. D’autres noms possibles pourraient être homepage ou main. Les décorateurs sont un sujet riche et intéressant pour les développeurs Python, donc si vous ne les connaissez pas encore, je vous recommande d’utiliser votre moteur de recherche préféré pour trouver un bon tutoriel.
  • La fonction index ne prend aucun argument. Cela peut sembler étrange si vous venez d’autres frameworks web où l’on attend souvent un objet requête (request) ou quelque chose de similaire. Nous verrons dans les exemples suivants comment accéder aux valeurs de la requête.
  • Les lignes suivantes exécutent notre application en utilisant le serveur de développement intégré en mode debug. L’instruction if est une convention Python courante qui garantit que l’application ne sera lancée que lorsque nous exécutons le script via python app.py, et ne sera pas lancée si nous essayons d’importer cette application depuis un autre fichier Python.
if __name__ == '__main__':
    app.run(debug=True)

Routes et requêtes

Pour l’instant, notre application Flask n’est pas très amusante, alors voyons les différentes façons d’ajouter un comportement plus intéressant à notre application web. Une méthode courante consiste à ajouter un comportement réactif, de sorte que notre application puisse examiner les valeurs présentes dans l’URL et les traiter.

Ajoutons une nouvelle route à notre application Hello Flask appelée hello. Cette nouvelle route affichera un message de bienvenue à la personne dont le nom apparaît dans l’URL :

from flask import Flask


app = Flask(__name__)

@app.route("/")
def hello():
    return "Hello, Flask!"

@app.route('/hello/<name>')
def hello(name):
    return 'Hello, %s' % name

if __name__ == '__main__':
    app.run(debug=True)

Voici une description possible du rendu dans le navigateur pour l’URL http://127.0.0.1:5000/hello/Charlie. Lorsque vous ouvrez cette URL dans votre navigateur, vous voyez une page blanche avec un message simple et personnalisé affiché en texte noir, centré en haut de la page. Le message dit :

Hello, Charlie

Ce message est généré dynamiquement par votre application Flask en utilisant la partie « Charlie » de l’URL pour personnaliser le contenu affiché à l’écran. C’est un exemple de contenu réactif en fonction des paramètres d’URL. Dans l’exemple précédent, la route que nous avons ajoutée spécifie un seul paramètre : name. Ce paramètre apparaît également dans la déclaration de la fonction en tant que seul argument. Flask associe automatiquement l’URL /hello/Charlie à la vue hello ; cela s’appelle le mapping. Il passe ensuite la chaîne de caractères « Charlie » à notre fonction de vue en tant qu’argument.

Que se passe-t-il si nous naviguons vers http://127.0.0.1:5000/hello/ sans spécifier de nom ? Comme vous pouvez le voir, le serveur de développement Flask renverra une réponse 404, indiquant que l’URL ne correspond à aucune route connue.

Lecture des valeurs de la requête

En plus de l'URL, des valeurs peuvent être transmises à votre application via la chaîne de requête (query string). Celle-ci est composée de paires clé-valeur arbitraires ajoutées à la fin de l'URL, après un point d'interrogation (?).

Tableau : liste d'urls avec des querystrings
URL Valeurs des arguments
/hello/?name=Charlie name: Charlie
/hello/?name=Charlie&favorite_color=green name: Charlie
favorite_color: green

Pour accéder à ces valeurs dans vos fonctions de vue, Flask fournit un objet request qui encapsule toutes les informations sur la requête HTTP en cours.

Dans l'exemple suivant, nous allons modifier notre vue hello pour qu'elle réagisse également aux noms passés via la chaîne de requête. Si aucun nom n'est spécifié (ni dans l'URL ni dans les paramètres), nous renverrons une erreur 404.

from flask import Flask, abort, request


app = Flask(__name__)

@app.route('/')
def index():
    return 'Hello, Flask!'

@app.route('/hello/<name>')
@app.route('/hello/')
def hello(name=None):
    if name is None:
        # Si aucun nom n’est spécifié dans l’URL, tenter de le récupérer depuis la chaîne de requête.
        name = request.args.get('name')
        if name:
            return 'Hello, %s' % name
    else:
        # Aucun nom n’a été spécifié dans l’URL ni dans la chaîne de requête.
        abort(404)

if __name__ == '__main__':
    app.run(debug=True, port=8000)

Comme vous pouvez le voir, nous avons ajouté un second décorateur de route à la vue hello. Flask permet d’associer plusieurs routes URL à une même fonction de vue. Puisque notre nouvelle route (/hello/) ne contient pas de paramètre name, nous devons adapter la signature de la fonction hello pour rendre ce paramètre optionnel. Cela se fait en lui assignant une valeur par défaut (None).

Le corps de la fonction a également été modifié pour vérifier si un nom est présent. Si aucun nom n’est fourni via l’URL dynamique, la fonction tente alors de le récupérer dans les paramètres de la chaîne de requête (request.args.get('name')). Si le nom est trouvé, un message de bienvenue est retourné. Dans le cas contraire, la fonction interrompt le traitement en retournant une erreur 404 – Page non trouvée.

Débogage d'applications Flask

l est inévitable que, tôt ou tard, nous introduisions un bug dans notre code. Puisque les bugs sont inévitables, le mieux que nous puissions espérer en tant que développeurs est d'avoir de bons outils pour les diagnostiquer et les corriger rapidement. Heureusement, Flask intègre un débogueur web extrêmement puissant.

Le débogueur de Flask permet d’inspecter l’état de votre application au moment même où une erreur se produit, éliminant ainsi le besoin d’ajouter des print ou des points d’arrêt manuellement.

Pour l’activer, il suffit de lancer l’application Flask en mode debug. Cela peut se faire de plusieurs manières, mais nous l’avons déjà mis en place via le code suivant :

if __name__ == '__main__':
    app.run(debug=True)

Pour tester cela, introduisons un bug dans l'application hello_flask en créant une faute de frappe. Ici, j'ai simplement supprimé le "e" final du nom de la variable :

 @app.route('/hello/<name>')
@app.route('/hello/')
def hello(name=None):
    if nam is None:
        # Si aucun nom n’est spécifié dans l’URL, tenter de le récupérer depuis la chaîne de requête.
        name = request.args.get('name')
        if name:
            return 'Hello, %s' % name
    else:
        # Aucun nom n’a été spécifié dans l’URL ni dans la chaîne de requête.
        abort(404)

Lorsque nous lançons le serveur de développement et tentons d’accéder à notre vue, la page de débogage s’affiche désormais. Cette liste de code s’appelle une traceback (ou pile d’appels) et elle est constituée de la pile des appels de fonctions imbriqués qui ont précédé l’erreur proprement dite. La traceback fournit en général un indice précieux sur ce qui a pu se passer. Tout en bas, on voit la ligne de code que nous avons intentionnellement mal écrite, accompagnée de l’erreur Python générée : une exception de type NameError, qui nous indique que nam n’est pas défini.

La véritable magie opère lorsque vous placez votre souris sur la ligne surlignée. À droite s’affichent deux petites icônes représentant un terminal et un fichier source. En cliquant sur l’icône Source Code, vous développez le code source autour de la ligne qui a généré l’erreur. C’est très utile pour replacer l’erreur dans son contexte.

L’icône Terminal est encore plus intéressante. En cliquant dessus, une console interactive apparaît avec l’invite Python standard. Cette console vous permet d’inspecter en temps réel la valeur des variables locales au moment où l’exception s’est produite. Essayez par exemple de taper name puis d’appuyer sur Entrée : si une valeur a été spécifiée dans l’URL, elle devrait s’afficher. On peut également consulter les arguments de la requête courante. Au fil des chapitres et de vos propres expérimentations, savoir diagnostiquer et corriger rapidement des bugs sera une compétence précieuse.
