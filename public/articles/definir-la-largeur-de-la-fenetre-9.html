Définir la largeur de la fenêtre

Parfois, les utilisateurs de veulent zoomer sur une page parce qu'ils ne peuvent pas lire le texte ou parce qu'ils veulent voir quelque chose de plus près. Ils ne peuvent pas le faire si les paramètres de la fenêtre de visualisation interdisent le zoom. C'est particulièrement problématique pour les personnes malvoyantes.

Configurez la balise méta viewport de manière à ce qu'elle soit la plus flexible possible. Évitez les paramètres restrictifs.

La balise meta de l'exemple 1 fonctionne pour la plupart des sites Web et des applications Web. C'est tout ce dont vous avez besoin pour configurer les paramètres de viewport afin de construire un site web flexible, adaptatif et réactif.

Exemple 1 La page utilise la largeur disponible de l'appareil comme largeur de la fenêtre de visualisation

<meta name="viewport" content="width=device-width, initial-scale=1">

Il existe des propriétés ou des valeurs spécifiques que vous devez éviter.

Définir la valeur de la propriété width à une valeur autre que device-width peut causer des problèmes. Si la largeur définie de la fenêtre de visualisation est supérieure à la largeur disponible sur l'écran, le contenu peut déborder, ce qui entraîne l'apparition de barres de défilement horizontales. L 'exemple 2 applique une largeur de fenêtre explicite.

Exemple 2 Mauvaise pratique : Largeur de la fenêtre de visualisation fixée à une valeur absolue.

<meta name="viewport" content="width=500, initial-scale=1">

maximum-scale vous permet de limiter le niveau de zoom maximum, qui est 10 par défaut dans la plupart des navigateurs. Si vous le définissez sur 1, vous désactiverez le zoom dans certains navigateurs (voir exemple 3.

Exemple 3 Mauvaise pratique : Zoom désactivé en réglant l'échelle maximale sur 1

<meta name="viewport" content="width=device-width, maximum-scale=1, initial-scale=1">

Le paramètre user-scalable définit si le zoom est autorisé. La valeur no ou 0 désactive le zoom, comme le montre l'exemple 4.

Exemple 4 Mauvaise pratique : Désactiver le zoom en réglant user-scalable sur "no"

<meta name="viewport" content="width=device-width, user-scalable=no, initial-scale=1">

Les utilisateurs doivent pouvoir personnaliser leur expérience de navigation en fonction de leurs préférences et de leurs besoins. Les navigateurs offrent différents paramètres et fonctionnalités pour y parvenir. La possibilité de définir des tailles de police plus grandes ou de zoomer sur la page est essentielle, mais de nombreux sites Web interdisent le zoom sur les appareils portables.

Avant que le responsive web design ne devienne une réalité, les sites web étaient conçus pour des écrans de grande taille, utilisant souvent des valeurs de largeur fixes comme 960px ou 1024px pour le corps ou le contenu principal de la page. Avec l'essor des smartphones et autres appareils portables, cela est devenu un problème car les largeurs de pixels des écrans de ces appareils étaient généralement beaucoup plus petites que 960px. Le bloc contenant initial, le rectangle dans lequel vit l'élément racine (<html>), a les dimensions de la fenêtre d'affichage. Si la page est plus grande que la fenêtre de visualisation, il peut en résulter une mise en page involontaire, un contenu tronqué et des limites de défilement désagréables. C'est pourquoi les navigateurs mobiles utilisent généralement une largeur fixe (généralement 980px à 1024px) pour le bloc initial. La présentation résultante est ensuite réduite pour s'adapter à l'espace disponible sur l'écran. Cela atténue les problèmes, mais cela signifie aussi que la taille des pixels CSS sur ces pages sera beaucoup plus petite, ce qui obligera les utilisateurs à zoomer.

Sur les sites Web réactifs, ce n'est pas un problème, car ils ont été conçus pour fonctionner correctement sur des interfaces d'affichage variées. Cependant, tu dois changer la largeur fixe du bloc contenant sur les appareils mobiles en une largeur relative aux dimensions de la fenêtre de visualisation pour qu'elle fonctionne bien avec les conceptions réactives. Tu peux le faire en utilisant la balise viewport meta, comme illustré dans l'exemple 1.

width=device-width définit la largeur de la fenêtre de visualisation en fonction de la largeur disponible de l'appareil.

initial-scale=1 garantit que le niveau de zoom par défaut est de 100 %. Il se peut que ce ne soit pas toujours la valeur par défaut dans tous les navigateurs. C'est pourquoi je recommande de le définir explicitement.

Le fait qu'une page soit responsive et optimisée pour les petits viewports ne signifie pas que les utilisateurs ne voudront pas zoomer. Adrian Roselli énumère plusieurs raisons pour lesquelles la possibilité de zoomer est essentielle dans son article « Don't Disable Zoom » :

  • Le texte peut être trop petit pour que l'utilisateur puisse le lire.
  • L'utilisateur peut vouloir voir plus de détails dans une image.
  • La sélection des mots à copier/coller peut être plus facile pour les utilisateurs lorsque le texte est plus grand.
  • L'utilisateur veut recadrer les éléments animés hors de la vue pour réduire les distractions.
  • Le développeur a fait un mauvais travail de conception responsive, et l'utilisateur doit zoomer juste pour utiliser la page.
  • Il y a un bug dans le navigateur qui fait que le niveau de zoom par défaut est bizarre.
  • Cela peut être déroutant pour les utilisateurs lorsque le navigateur interprète un geste de pincement/écartement comme quelque chose d'autre.

Les sites web qui désactivent le zoom sont un problème répandu. Selon le chapitre sur l'accessibilité de l'Almanach du Web 2022, 23 % des pages d'accueil de bureau et 28 % des pages d'accueil mobiles tentent de désactiver le zoom. L'auteur du rapport utilise le terme tentative, car certains navigateurs, comme Safari sur iOS ou Samsung Internet sur Android, ignorent les propriétés maximum-scale=1 et user-scalable=no. Chrome et Firefox ne le font pas, mais les utilisateurs peuvent forcer le zoom dans les paramètres de leur navigateur. Dans Firefox, trouve les paramètres du navigateur, sélectionne "Accessibilité" et active « Zoom sur tous les sites web ». Dans Chrome, trouve les paramètres du navigateur, sélectionne "Accessibilité" et coche « Forcer l'activation du zoom ».

Raisons justifiées de désactiver le zoom

Sur le site web moyen, il n'y a pas de bonne raison de désactiver le zoom. Il en va de même pour les sites web de type app qui ressemblent à des apps natives. Il existe de rares exceptions où les gestes pour zoomer interféreraient avec les fonctionnalités du site web. Un exemple serait un site qui ne contient qu'une carte interactive. Si c'est le cas, il peut être acceptable de désactiver la fonction de zoom native, mais vous devez fournir une solution alternative personnalisée .
