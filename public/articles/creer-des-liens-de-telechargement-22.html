Créer des liens de téléchargement

La polyvalence de l'élément <a> exige qu'il communique, avant que l'utilisateur ne clique dessus, son but, sa fonctionnalité et l'action qu'un clic s'exécutera. Lorsque vous établissez un lien vers un fichier au lieu d'un document, les utilisateurs doivent connaître à l'avance le format et la taille du fichier, ainsi que la façon dont ils y accéderont. Les changements inattendus de contexte peuvent être irritants, et le téléchargement de fichiers volumineux peut avoir des conséquences financières pour les utilisateurs.

Les liens de l'exemple 1 répondent à des questions essentielles :

  • Comment puis-je accéder au fichier ?
  • Quelles sont les informations contenues dans le fichier ?
  • Quel est son format ?
  • Quelle est sa taille ?

Exemple 1 – Liens vers des fichiers

<!-- Lien vers fichier pdf -->
<a href="/menu.pdf">
  Voir notre menu (PDF, 1.2MB)
</a>

<!-- télécharger fichier pdf -->
<a href="/report.pdf" download>
  Télécharger notre rapport 2022 (PDF, 29MB)
</a>

Pour les liens de téléchargement, vous pouvez également fournir un style différent ou supplémentaire ; par exemple, vous pouvez utiliser une icône qui symbolise un téléchargement, comme le montrent l'exemple 2.

Exemple 2 – Utilisation de l'attribut download comme crochet en CSS

[download]::after {
  content: "";
  background: url('../icons/download.svg') no-repeat center;
  block-size: 1em;
  display: inline-block;
  inline-size: 1em;
}

Vous ppouvez aussi utiliser l'attribut download pour renommer un fichier. Au lieu du nom de fichier original, le nom fourni comme valeur de l'attribut download sera utilisé lors du téléchargement, comme le montre l'exemple 3.

Exemple 3 – Un lien vers un fichier avec un nom de fichier personnalisé

<a href="/hdj588pwW-1312d-oek92x.xls" download="ventes-q1-2023.xls">
  ventes 1er trimestre 2023 (Microsoft Excel, 0.3MB)
</a>

Le texte d'un lien doit décrire son objectif pour permettre aux utilisateurs de le distinguer des autres liens d'une page Web et de déterminer s'ils doivent le suivre. Vous pouvez aider les utilisateurs à comprendre ce qui se passera lorsqu'ils cliqueront sur un lien vers un fichier en incluant des actions spécifiques dans le texte du lien. Vous pouvez utiliser des expressions comme « voir » ou « télécharger », en fonction de l'objectif, mais toujours en combinaison avec la cible, afin d'éviter les textes de lien génériques. Les exemples 4 et 5 montrent deux façons différentes de créer un lien vers une image.

Exemple 4 – Mauvaise pratique : Un lien vers une image et un lien de téléchargement d'une image avec un texte générique.

<a href="/images/flower.jpg">
  Voir
</a>

<a href="/images/flower.jpg" download>
  Télécharger
</a>

L'utilisateur sait à quoi s'attendre en lisant le texte du lien dans l'exemple 5.

Exemple 5 – Un lien vers une image et un lien de téléchargement pour une image avec un texte spécifique.

<a href="/images/flower.jpg">
  Voir la photo de la fleur
</a>

<a href="/images/flower.jpg" download>
  Télécharger la photo de la fleur
</a>

Les utilisateurs peuvent préférer connaître le type d'un fichier lié pour éviter les téléchargements inutiles si leur système d'exploitation ou leur logiciel ne prend pas en charge ce type de fichier, comme illustré dans l'exemple 6.

Exemple 6 – Un lien vers un type de fichier potentiellement non pris en charge

<a href="/files/logo.ai" download>
   Télécharger le logo (Adobe Illustrator)
</a>

Certains utilisateurs évitent volontairement certains formats de fichiers, comme les PDF, souvent jugés inaccessibles. C’est d’ailleurs un enjeu reconnu, avec la norme ISO 14289 (PDF/UA), conçue pour rendre les PDF plus compatibles avec les technologies d’assistance.

Au lieu de texte, vous pouvez utiliser des icônes pour communiquer le type de fichier, comme le montre l'exemple 7. En fonction de votre public, cette méthode peut fonctionner ou non. Le plus grand inconvénient peut être que vos utilisateurs ne comprennent pas les icônes spécifiques. Si vous souhaitez jouer la carte de la sécurité, je vous recommande d'utiliser du texte.

Exemple 7 – Utilisation d'un sélecteur d'attribut et d'un pseudo-élément pour afficher des icônes en fonction du type de fichier lié.

a[href$=".pdf"]::after {
  background-image: url(../icons/pdf.svg);
}

a[href$=".tiff"]::after {
  background-image: url(../icons/tiff.svg);
}

Pour certaines personnes, il est important de connaître la taille du fichier, en particulier celles qui ont des connexions lentes ou qui n'ont pas de plan de données illimité. Vous pouvez inclure une taille approximative à côté du type de fichier entre parenthèses après le texte du lien, comme le montre l'exemple 8.

Exemple 8 – Un lien vers un fichier volumineux

<a href="/images/garden.tiff">
  Version haute résolution de l'image du jardin (TIFF, 6MB)
</a>

La façon dont les liens de fichiers sont traités varie selon le navigateur, les paramètres de l'utilisateur, les applications installées, le type de fichier et d'autres facteurs. Un lien vers une image s'ouvre généralement dans la même fenêtre ou le même onglet, tandis qu'un lien vers un document peut prompt à un téléchargement ou ouvrir automatiquement une application externe.

Vous pouvez forcer le téléchargement en utilisant l'attribut de téléchargement lorsque vous placez le lien. Cependant, n'oubliez pas que la présence de l'attribut ne transmet aucune information aux utilisateurs, y compris aux utilisateurs de lecteurs d'écran, et que le texte du lien doit donc se charger de cette tâche, comme illustré dans l'exemple 1. L'utilisation de l'attribut download dépend du contexte. Si vous établissez un lien vers un fichier que le navigateur prend en charge, comme un fichier HTML, l'ajout de l'attribut faciliterait le téléchargement pour les utilisateurs. D'un autre côté, l'utilisation de l'attribut peut limiter le choix des utilisateurs quant à la façon d'accéder à un fichier .

Important : L'attribut download ne fonctionne que pour les URL de même origine ou les schémas blob: et data:.
