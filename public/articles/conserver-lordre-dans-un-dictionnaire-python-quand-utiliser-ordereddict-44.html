Conserver l’ordre dans un dictionnaire Python : quand utiliser OrderedDict ?

Vous devez générer un fichier JSON pour une API tierce qui exige un ordre strict des champs. Vous pensez naturellement à utiliser un dictionnaire Python… mais comment vous assurer que les champs apparaîtront dans le bon ordre lors de la sérialisation ?

Utiliser OrderedDict pour garantir l’ordre

Pour contrôler l’ordre des éléments dans un dictionnaire, Python propose la classe OrderedDict dans le module collections. Cette structure de données préserve exactement l’ordre d’insertion des paires clé-valeur, ce qui est utile pour l’itération ou la sérialisation (par exemple en JSON). Voici un exemple simple :

from collections import OrderedDict


d = OrderedDict()

d['a'] = 1
d['b'] = 2
d['c'] = 3
d['d'] = 4
d['e'] = 5

for k, v in d.items():
    print(k, v)

Lorsqu'on sérialise ce dictionnaire avec json.dumps(), l’ordre est respecté :

import json


print(json.dumps(d))

Ce comportement est particulièrement utile si vous avez besoin de générer un mapping structuré dans un format externe (JSON, XML, CSV…) où l’ordre des champs est important.

Fonctionnement interne

Un OrderedDict repose sur une liste doublement chaînée pour maintenir l’ordre des clés. Quand un élément est ajouté, il est placé à la fin de cette liste. Si vous modifiez la valeur d’une clé déjà présente, son positionnement ne change pas.

Inconvénient : le coût mémoire

Un OrderedDict est plus lourd qu’un dictionnaire classique. Par exemple :

import sys
from collections import OrderedDict


d1 = {'a': 1, 'b': 2, 'c': 3}
d2 = OrderedDict(d1)

print(sys.getsizeof(d1))  # ~232 octets
print(sys.getsizeof(d2))  # ~456 octets

Résultat : un OrderedDict peut consommer deux fois plus de mémoire qu’un dict. Si vous chargez des dizaines de milliers d’objets (ex. : parsing de CSV), ce surcoût peut devenir un facteur limitant.

À savoir : les dict sont ordonnés depuis Python 3.7

Depuis Python 3.7, les dictionnaires standards (dict) conservent déjà l’ordre d’insertion des éléments. Ce comportement, apparu en Python 3.6 comme un détail d’implémentation, est devenu une caractéristique officielle du langage. Autrement dit : dans une version moderne de Python, le code ci-dessous produira un JSON dans le même ordre que l’insertion.

d = {'a': 1, 'b': 2, 'c': 3}
print(json.dumps(d))

Si vous utilisez Python 3.7 ou supérieur (ce qui est très probable aujourd’hui), et que vous n’avez pas besoin de méthodes spécifiques à OrderedDict comme move_to_end() ou popitem(last=False), alors un simple dict est suffisant et plus léger en mémoire.

Méthodes avancées de OrderedDict

Au-delà de la simple conservation de l’ordre, OrderedDict offre des méthodes spécifiques pour manipuler précisément la position des éléments. move_to_end(key, last=True) : déplace une clé existante à la fin (par défaut) ou au début (last=False) du dictionnaire.

from collections import OrderedDict

d = OrderedDict()
d['a'] = 1
d['b'] = 2
d['c'] = 3

print("Avant :", list(d.items()))

# On déplace 'a' à la fin
d.move_to_end('a')

print("Après move_to_end('a') :", list(d.items()))

# On déplace 'c' au début
d.move_to_end('c', last=False)

print("Après move_to_end('c', last=False) :", list(d.items()))

popitem(last=True) : supprime et retourne une paire clé-valeur. Par défaut, elle retire le dernier élément (last=True), mais on peut retirer le premier en passant last=False.

from collections import OrderedDict

d = OrderedDict([('a', 1), ('b', 2), ('c', 3)])

# Retire le dernier élément
last_item = d.popitem()
print("popitem() =>", last_item)
print("Reste :", list(d.items()))

# Retire le premier élément
first_item = d.popitem(last=False)
print("popitem(last=False) =>", first_item)
print("Reste :", list(d.items()))

Ces méthodes permettent de gérer des files d’attente, des caches ou des structures où l’ordre et la manipulation fine des éléments sont importants.
