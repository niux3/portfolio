Gérer un environnement virtuel avec Python, pip et Pipenv

Lorsque vous développez plusieurs projets Python, chacun peut nécessiter des bibliothèques différentes, voire des versions spécifiques de ces bibliothèques. Pour éviter les conflits, Python permet de créer des environnements virtuels : des espaces isolés dans lesquels vous installez uniquement les dépendances dont votre projet a besoin. Ainsi, votre système reste propre et vous maîtrisez exactement ce que votre projet utilise.

Créer un environnement virtuel avec venv

La première étape consiste à créer un environnement virtuel dans le répertoire de votre projet. Cela va générer un dossier contenant une version indépendante de l’interpréteur Python ainsi que de pip.

$python3 -m venv .venv

Ici, .venv est le nom du dossier qui contiendra l’environnement virtuel. Vous pouvez l’appeler autrement, mais .venv est une convention largement adoptée. Cette commande ne fait qu’installer une structure de fichiers, elle ne lance pas l’environnement.

Activer l’environnement virtuel

Avant d’utiliser votre environnement, vous devez l’activer. L’activation consiste à modifier temporairement ta session pour que Python et pip pointent vers l’environnement virtuel, et non vers les versions globales du système.

Selon le système :

  • GNU/Linux - MacOS :
source .venv/bin/activate
  • Microsoft Windows : 
.venv\Scripts\activate.bat

Une fois activé, vous verrez le nom de l’environnement s’afficher dans ton terminal, ce qui confirme que vous travaillez bien dans ce contexte isolé.

Installer des bibliothèques avec pip

Maintenant que votre environnement est actif, vous pouvez installer les bibliothèques dont votre projet a besoin en utilisant pip.

pip install requests flask

Ces bibliothèques seront installées uniquement dans l’environnement virtuel .venv, et n’auront aucun impact sur les autres projets ou sur votre système.

Installer une version spécifique d’un module

Lorsque vous travaillez sur un projet, il est parfois essentiel d’installer une version précise d’un module pour garantir la compatibilité ou éviter une régression.

pip install numpy==1.21.0

Quelques variantes utiles :

  • == : exactement cette version
  • >= : au moins cette version
  • <= : au plus cette version
  • != : toute version sauf celle-ci
  • ~= : compatible avec (ex : requests~=2.25 installe la dernière 2.25.x)

avec requirements.txt : Vous pouvez aussi fixer la version dans le fichier requirements.txt :

Django==4.2.3
requests>=2.26,<3.0

Puis installer toutes les dépendances :

pip install -r requirements.txt

Gérer les dépendances

Quand vous travaillez sur un projet, il est important de pouvoir facilement partager les bibliothèques utilisées. Pour cela, on utilise un fichier requirements.txt qui liste toutes les dépendances.

Pour sauvegarder les dépendances :

pip freeze > requirements.txt

Pour réinstaller les mêmes dépendances ailleurs :

pip install -r requirements.txt

Ainsi, n’importe qui (ou vous-même dans quelques mois) pourra recréer exactement le même environnement.

Désactiver l’environnement

Lorsque vous avez fini de travailler sur votre projet, vous pouvez désactiver l’environnement virtuel. Cela remet le terminal dans son état initial, en sortant de l’environnement isolé.

deactivate

Votre interpréteur Python redevient alors celui du système.

Supprimer un environnement virtuel

Si vous n’avez plus besoin de l’environnement (par exemple, si vous supprimez le projet), vous pouvez simplement supprimer le dossier .venv. Cela ne supprime que l’environnement, pas votre projet ou vos fichiers de code.

Selon le système :

  • GNU/Linux - MacOS :
rm -rf .venv
  • Microsoft Windows : 
rd /s /q .venv

Utiliser Pipenv pour simplifier la gestion des environnements et des dépendances

Bien que venv et pip soient des outils puissants, l'utilisation de Pipenv peut grandement simplifier la gestion des environnements virtuels et des dépendances. Pipenv combine les fonctionnalités de pip et virtualenv en un seul outil, offrant une expérience plus fluide et moins sujette aux erreurs.

La documentation officielle de Python recommande l'utilisation de Pipenv pour la gestion des dépendances des applications.

Dans le Python Packaging User Guide, il est indiqué que Pipenv est l'un des outils reconnus pour créer des fichiers de verrouillage (lock files), qui contiennent les versions exactes de tous les paquets installés dans un environnement, afin d'assurer la reproductibilité. Cela est particulièrement utile pour les projets collaboratifs où il est essentiel que tous les contributeurs utilisent les mêmes versions de dépendances.

De plus, le guide Managing Application Dependencies présente un tutoriel détaillé sur l'utilisation de Pipenv pour gérer les dépendances d'une application. Il y est mentionné que, bien que pip soit souvent suffisant pour un usage personnel, Pipenv est recommandé pour les projets collaboratifs, car c'est un outil de plus haut niveau qui simplifie la gestion des dépendances pour les cas d'utilisation courants.

En résumé, Pipenv est recommandé par la documentation officielle de Python pour la gestion des dépendances des applications, en particulier dans des contextes collaboratifs où la reproductibilité et la simplicité de gestion sont essentielles.

Installation de Pipenv

Installez Pipenv en utilisant pip :

pip install --user pipenv

Note: Il est recommandé d'utiliser l'option --user pour éviter les problèmes de permissions et les conflits avec les paquets système.

Créer un nouveau projet avec Pipenv

Initialiser un projet

Dans le répertoire de votre projet, exécutez :

pipenv shell

Note: Cela créera un fichier Pipfile qui gérera les dépendances de votre projet.

Installer des paquets

Pour installer une bibliothèque, par exemple requests :

pipenv install requests

Désactiver l'environnement virtuel : Pour sortir de l'environnement virtuel :

exit

Gérer les dépendances de développement : Pour installer des paquets nécessaires uniquement pour le développement, utilise l'option --dev :

pipenv install --dev pytest

Vérifier les dépendances : Pour vérifier les vulnérabilités de sécurité dans les dépendances :

pipenv check

Avantages de Pipenv

  • Gestion simplifiée : Unifie la gestion des environnements virtuels et des dépendances.
  • Reproductibilité : Le fichier Pipfile.lock assure que tous les contributeurs utilisent les mêmes versions de paquets.
  • Sécurité : Intègre des outils pour vérifier les vulnérabilités dans les dépendances.
  • Compatibilité : Fonctionne sur Windows, macOS et Linux.
