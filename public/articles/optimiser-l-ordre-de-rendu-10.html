L'en-tête d'un document peut contenir divers éléments qui servent à des fins différentes : balises méta, scripts, liens vers d'autres ressources, et plus encore. Ils peuvent être placés dans n'importe quel ordre, mais certains éléments doivent passer avant d'autres pour garantir de bonnes performances de chargement. Un chargement inefficace des ressources empêche les utilisateurs d'obtenir des informations rapidement, voire de les obtenir tout court.

L'expert en performances Web Harry Roberts suggère un ordre spécifique des éléments dans le site <head> pour garantir la meilleure stratégie de chargement possible, comme le montre l'exemple 1.

Exemple 1 L'ordre idéal des éléments dans le <head>

<head>
  <!-- Character encoding -->
  <meta charset="UTF-8">

  <!-- Viewport meta tag -->
  <meta name="viewport" content="width=device-width, initial-scale=1">

  <!-- CSP headers -->
  <meta http-equiv="Content-Security-Policy" content="upgrade-insecure-requests">

  <!-- Page title -->
  <title>Magasin vêtement Renaud</title>

  <!-- preconnect -->
  <link rel="preconnect" href="#" />

  <!-- Asynchronous JavaScript -->
  <script src="" async></script>

  <!-- CSS that includes @import -->
  <style>
    @import "file.css";
  </style>

  <!-- Synchronous JavaScript -->
  <script src=""></script>

  <!-- Synchronous CSS -->
  <link rel="stylesheet" href="#">

  <!-- preload -->
  <link rel="preload" href="#" />

  <!-- Deferred JavaScript -->
  <script src="" defer></script>
  <script src="" type="module"></script>

  <!-- prefetch / prerender -->
  <link rel="prefetch" href="#" />
  <link rel="prerender" href="#" />

  <!-- divers -->
  <meta name="description" content="">
</head>

Il existe différents domaines au sein de la conception et du développement web, comme l'accessibilité, la convivialité, l'expérience utilisateur, la performance et la sécurité. Ils se concentrent tous sur des choses différentes, mais ils ne s'excluent pas mutuellement. L'accessibilité, par exemple, recoupe tous ces domaines. Améliorer l'accessibilité des champs de formulaire peut se traduire par une meilleure expérience utilisateur pour tout le monde. Si un site web se charge lentement, cela affecte l'expérience des utilisateurs et l'accessibilité. Un site Web qui se charge trop longtemps ou pas du tout sur une connexion lente n'est pas accessible. Concevoir et construire des sites Web accessibles signifie créer des expériences inclusives sans barrières qui empêchent les gens d'interagir avec le Web. Ces obstacles comprennent les handicaps physiques, temporaires et situationnels, ainsi que les restrictions socio-économiques concernant le matériel, la bande passante et la vitesse.

Le fait de respecter l'ordre des éléments dans le site <head> a une incidence sur les performances en général, mais aussi sur le rendu d'éléments spécifiques qui peuvent contenir des informations essentielles pour les utilisateurs de technologies d'assistance. Le HTML est analysé ligne par ligne, ce qui signifie qu'un navigateur ne sait pas que la ligne 4 existe alors que la ligne 3 n'a pas fini d'être analysée. Si quelque chose bloque le rendu au début d'un document, les lignes suivantes doivent attendre que le navigateur ait fini d'analyser les lignes précédentes. L'ordre correct des éléments dans le site <head> est donc crucial pour les performances et l'accessibilité du Web.

Les experts en performance suggèrent plusieurs règles et optimisations, notamment :

  • Si quelque chose n'a pas besoin d'être dans le site <head>, supprimez-le ou mettez-le dans le site <body>. Cela inclut les scripts de faible priorité, les redirections ou toute charge utile inutile.
  • Auto-hébergez autant que possible et ne comptez pas sur des réseaux de diffusion de contenu (CDN) tiers. Harry Roberts explique pourquoi dans son article « Self-Host Your Static Assets ».
  • Validez votre code HTML, car les éléments non valides dans le <head> peuvent causer des problèmes de performance.
  • Les métadonnées sur la page, comme le codage des caractères et les informations sur le viewport, passent en premier.
  • Rien de ce qui bloque le rendu ne doit venir avant le <title>.
  • Le JavaScript synchrone vient avant le CSS, parce que le CSS bloque l'exécution du JavaScript suivant.
  • Évitez @import dans les CSS.
  • Les balises méta de référencement et sociales passent en dernier.

Harry Roberts a créé un fichier CSS appelé ct.css, également disponible sous forme de bookmarklet, que tu peux utiliser pour effectuer des tests sur tes éléments <head>.
