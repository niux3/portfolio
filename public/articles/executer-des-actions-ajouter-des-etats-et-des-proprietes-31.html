Exécuter des actions : ajouter des états et des propriétés

Lorsque les utilisateurs du lecteur d'écran utilisent des boutons pour contrôler d'autres éléments de la page ou les paramètres du site, ces boutons doivent fournir autant d'informations que possible. Il peut s'agir du type de bouton, du type d'élément associé ou de l'état d'un bouton ou d'un élément contrôlé. Si ces informations manquent, il est beaucoup plus compliqué - voire impossible - de savoir si l'utilisateur a bien activé le bouton et ce qui se passe lorsqu'il le fait.

Un bouton qui fait basculer la visibilité d'un autre élément doit indiquer si l'élément est développé, comme le montre dans l'exemple 1.

Exemple 1 – L'attribut aria-expanded du bouton communique si la navigation est visible.

<nav>
  <button aria-expanded="false" aria-controls="main_nav">
    Navigation
  </button>

  <ul id="main_nav" aria-hidden="true">
    <li>Accueil</li>
    <li>À propos</li>
    <li>Contact</li>
  </ul>
</nav>

<style>
  [aria-expanded="false"] + ul {
    display: none;
  }
</style>

<script>
const button = document.querySelector("button");
const menu = document.getElementById("main_nav");

button.addEventListener("click", () => {
  const isExpanded = button.getAttribute("aria-expanded") === "true";
  button.setAttribute("aria-expanded", String(!isExpanded));
  menu.setAttribute("aria-hidden", String(isExpanded));
});
</script>

Remarque: - aria-expanded doit se trouver sur le bouton et non sur l'élément extensible. - La liste est masquée, en fonction de la valeur de l'attribut. - L'événement de clic sur le bouton fait basculer la valeur de l'attribut aria-expanded et aria-hidden.

Un bouton qui active ou désactive un paramètre doit communiquer s'il est actif, comme le montre l'exemple 2.

Exemple 2 – Un bouton communiquant son état enfoncé

<button type="button" aria-pressed="true">
Ajouter aux favoris
</button>

<script>
const button = document.querySelector("button");

button.addEventListener("click", (e) => {
  const isPressed = button.getAttribute("aria-pressed") === "true";
  button.setAttribute("aria-pressed", !isPressed);
});
</script>

Un bouton qui fait basculer un paramètre doit indiquer s'il est actif. L'exemple 3 montrent un bouton d'interrupteur qui communique son état à l'aide de l'attribut aria-checked.

Exemple 3 – Un bouton d'interrupteur

<button role="switch" aria-checked="false">Se souvenir de moi</button>

<script>
const button = document.querySelector("button");

button.addEventListener("click", (e) => {
  const isChecked = button.getAttribute("aria-checked") === "true";
  button.setAttribute("aria-checked", !isChecked);
});
</script>

Un bouton peut communiquer son état et le type d'élément qu'il contrôle, comme le montre l'exemple 4.

Exemple 4 – Un bouton contrôlant un menu

<button
  type="button"
  aria-haspopup="menu"
  aria-expanded="false"
  id="button_settings"
>
  Paramètres
</button>

<ul role="menu" aria-labelledby="button_settings" aria-hidden="true" hidden>
  <li role="none">
    <button role="menuitem">Imprimer</button>
  </li>
  <li role="none">
    <button role="menuitem">Sauvegarder</button>
  </li>
</ul>

<style>
  [aria-expanded="true"] + ul {
    display: block;
  }
</style>

<script>
const button = document.querySelector("button");
const menu = button.nextElementSibling;

button.addEventListener("click", () => {
  const isExpanded = button.getAttribute("aria-expanded") === "true";
  button.setAttribute("aria-expanded", String(!isExpanded));
  menu.setAttribute("aria-hidden", String(isExpanded));
});
</script>

De nombreux attributs de la spécification ARIA fournissent aux éléments des états ou des propriétés. Lorsque vous construirez des scripts JavaScript personnalisés, vous utiliserez souvent certains d'entre eux. La propriété arial-label, par exemple, donne à un élément un nom accessible, tandis que l'état aria-hidden supprime un élément de l'arbre d'accessibilité.

Cet article se concentre sur quatre attributs couramment utilisés avec les boutons.

L'état élargi

Vous utilisez l'attribut aria-expanded sur un élément bouton pour indiquer si un élément de regroupement qu'il contrôle est développé ou réduit.

Note : L'élément contrôlé peut être à peu près n'importe quel élément, mais il s'agit généralement d'un élément de regroupement comme <div>, <p>, ou <ul>.

Dans l'exemple 1, vous pouvez voir que le bouton possède l'attribut et communique que l'élément associé est développé. Les utilisateurs doivent comprendre, à partir du texte du bouton, quel élément il contrôle. C'est pourquoi vous devez éviter les textes génériques comme « show/hide ». L'attribut peut être utile pour les navigations à la volée, pour les sous-menus dans les navigations imbriquées et pour les scripts Javascript de divulgation.

Le rôle du bouton est de communiquer si l'élément contrôlé est développé. Vous devez suivre trois règles essentielles lorsque voius l'appliquez :

  1. Vous définissez l'attribut sur l'élément qui effectue le contrôle (le bouton), et non sur l'élément contrôlé (le groupe).
  2. La simple présence de l'attribut ne suffit pas, vous devez lui attribuer la valeur « vrai » ou « faux ».
  3. L'attribut doit être présent et défini avant que l'utilisateur n'interagisse avec le bouton. Si vous donnez la valeur « false » à l'attribut, cela signifie que l'élément contrôlé est réduit. Si vous ne définissez pas l'attribut, le bouton ne contrôle rien.

Comme vous ppouvez le voir dans l'exemple 1, le bouton possède un autre attribut ARIA : aria-controls.

La propriété des contrôles

L'attribut aria-controls identifie l'élément que le bouton contrôle. La valeur est une liste d'une ou plusieurs références id. Lorsque l'attribut est présent, un lecteur d'écran peut identifier une relation entre un bouton et un autre élément. JAWS n'annonce pas automatiquement cette relation, mais vous pouvez utiliser le raccourci JAWSKEY + ALT + M pour accéder directement à l'élément contrôlé.

Pour les widgets de divulgation, cet attribut n'est pas bien supporté (JAWS est le seul lecteur d'écran qui l'utilise), et on ne sait pas quelle importance les développeurs de lecteurs d'écran prévoient de lui accorder. Pour NVDA, il s'agit d'une question ouverte depuis 2018; une discussion ouverte dans le dépôt GitHub ARIA date de 2019. C'est à vous de décider si vous l'utilisez ou non. En attendant une recommandation officielle pour ou contre, je l'utilise.

État pressé

L'attribut aria-pressed indique l'état actuel « enfoncé » des boutons à bascule.

Un bouton à bascule est semblable à une case à cocher, mais pas tout à fait la même chose. Outre le style, la différence la plus importante est qu'une case à cocher ne transmet qu'un état (coché/décoché/mixte), alors qu'un bouton effectue une action. Lorsque les utilisateurs cliquent sur un bouton à bascule, ils s'attendent à ce qu'il se passe quelque chose. Le fait d'appuyer sur le bouton dans l'exemple 2 fait basculer la valeur de l'attribut aria-pressed et modifie l'état du bouton Ajouter aux favoris. Les modifications côté client de ce type nécessitent que vous travaillez dans un environnement qui s'appuie sur JavaScript, car l'état pressé doit changer au moment du clic. Si ce n'est pas le cas et que vous voulez améliorer le contrôle progressivement, utilise plutôt une case à cocher.

État vérifié

L'attribut aria-checked indique l'état actuel « coché » des cases à cocher, des boutons radio et autres widgets.

Vous n'êtes pas censé utiliser aria-checked sur un bouton ayant le rôle button, mais vous pouvez l'utiliser sur un interrupteur (voir l'exemple 3). Un interrupteur est un type de case à cocher qui représente des valeurs activé/désactivé au lieu de valeurs coché/décoché/mixé. Il offre à peu près la même fonctionnalité qu'une case à cocher ou un bouton à bascule, mais vous pouvez les distinguer pour les lecteurs d'écran d'une manière cohérente avec son apparence à l'écran. Les interrupteurs sont un modèle problématique en termes d'expérience utilisateur. Si vous décidez d'utiliser des interrupteurs, testez-les soigneusement avec les utilisateurs, y compris ceux qui utilisent des lecteurs d'écran.

propriété haspopup

L'attribut aria-haspopup indique qu'un bouton contrôle un autre élément pop-up interactif. La plupart des lecteurs d'écran annoncent également le type d'élément pop-up . L'attribut prend en charge sept valeurs : true, false, menu, dialog, grid, listbox et tree, indiquant que l'élément référencé a le rôle correspondant. true est identique à menu.

Selon le logiciel de lecture d'écran que vous utilisez, si vous focalisez le bouton de l'exemple 4, il annoncera quelque chose comme « Réglages, bouton, menu » (JAWS) ou « Réglages, bouton pop-up, menu pop-up » (VoiceOver). La valeur que vous utilisez doit tenir sa promesse : si la valeur est menu, le rôle de l'élément contrôlé doit également être menu et fonctionner en conséquence. JAWS, par exemple, annonce également des instructions appropriées lorsqu'un attribut ayant une valeur spécifique est présent. Pour true et menu, il annonce : « Appuyer sur Space pour activer le menu. Puis navigue avec les touches fléchées. » Pour listbox, tree et grid: « Pour activer, appuie sur Entrée ».

Les valeurs true et menu sont bien supportées par tous les lecteurs d'écran. Cependant, TalkBack et Narrator ne prennent pas en charge les grilles, les boîtes de dialogue, les boîtes de liste ou les arbres.
