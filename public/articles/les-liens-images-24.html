Une adresse incorrecte ou des descriptions manquantes d'images liées donnent lieu à des liens confus ou brisés, ce qui peut empêcher les utilisateurs de lecteurs d'écran de comprendre l'objectif du lien.

Utilise l'attribut alt de l'image pour fournir l'étiquette ou le nom accessible du lien. L'exemple 1 montre un logo lié dans l'en-tête d'une page. La description de l'image est « Page d'accueil » parce que c'est là que le lien pointe. Une erreur courante consiste à utiliser une description telle que  «Logo de la société ». Cette image est fonctionnelle et doit donc décrire l'objectif du lien, et non le contenu de l'image.

Exemple 1 – Le texte alternatif du logo lié sert de texte de lien

<header>
  <a href="/">
    <img src="/images/logo.svg" alt="Home page">
  </a>
</header>

Au lieu d'un img, vous pouvez aussi utiliser un graphique vectoriel scalable (SVG ) avec un élément <title>. Pour la prise en charge inter-navigateurs, vous devriez utiliser aria-labelledby sur le SVG et créer une référence au titre. Note que certains navigateurs affichent le contenu de l'élément <title> dans une infobulle lorsque vous survolez le SVG, comme le montre l'exemple 2.

Exemple 2 – Le titre du SVG sert de texte au lien de téléchargement

<a href="/report.pdf" download>
  <svg aria-labelledby="save_title" role="img">
    <title id="save_title">Save</title>
    <path d="…"/>
  </svg>
</a>

Vous pouvez également retirer le graphique de l'arbre d'accessibilité en définissant un attribut alt vide sur img ou aria-hidden="true" sur le SVG. Si vous faites cela, le lien a toujours besoin d'une alternative textuelle, que vous pouvez fournir avec aria-label ou aria-labelledby sur le lien lui-même, comme le montre l'exemple 3.

Exemple 3 – L'attribut aria-label étiquette le lien

<a href="https://mastodon.social" aria-label="Mastodon">
  <img src="/images/mastodon-logo.svg" alt="">
</a>

<a href="https://mastodon.social" aria-label="Mastodon">
  <svg aria-hidden="true">
    <path d="…"/>
  </svg>
</a>

Si vous n’utilisez pas l’image pour attribuer un nom accessible au lien ou pour l’étiqueter d’une autre manière, un lecteur d’écran annoncera l’URL, une partie de celle-ci, une formule du type « image non étiquetée », ou encore le nom du fichier de l’image.

Les images sur le web peuvent être grossièrement classées en trois catégories : informatives, décoratives et fonctionnelles.

Les images informatives représentent des informations liées au contenu principal d’une page ou d’une section. Décrivez brièvement ce type d’images en expliquant leur signification ou le contenu qu’elles affichent. La longueur de la description dépend des informations et du contexte que vous souhaitez transmettre. L’exemple 4, vous trouverez un exemple d’image informative ainsi que sa description.

Exemple 4 – Description d'une image informative

<img src="images/karlskirche.jpg" alt="L'église Saint-Charles de Vienne, un édifice baroque contenant un dôme et deux colonnes.">

Les images décoratives servent uniquement à l’aspect visuel et n’apportent que peu, voire aucune information en termes de contenu. Elles n’ont donc pas besoin de texte alternatif. Dans l’exemple 5, vous trouverez un exemple d’image décorative accompagné de sa description.

Exemple 5 – L'attribut alt blank cache l'image décorative aux lecteurs d'écran.

<img src="images/headphones.jpg" alt="">

Dans certaines situations, les images décoratives ne contribuent pas directement à la compréhension du contenu, mais elles peuvent transmettre des émotions. Dans ce cas, lorsque vous souhaitez instaurer une atmosphère particulière, il peut être pertinent de décrire ces images décoratives.

Les images fonctionnelles sont des images intégrées dans des liens ou des boutons, dont le texte alternatif fait office de libellé pour le lien ou le bouton. Avec ce type d’image, vous ne décrivez pas le contenu visuel, mais plutôt l’action que l’image initie.

Selon le rapport WebAIM Million 2024, issu d’une évaluation automatisée annuelle de l’accessibilité du top 1 million de sites web, 44,6 % des sites testés contenaient des liens vides.

Il existe au moins deux causes principales à ces problèmes. La première réside dans la présence de liens littéralement vides, c’est-à-dire ne contenant aucun contenu, comme le montre l’exemple 6.

Avec les images informatives, vous expliquez le contenu, tandis qu’avec les images fonctionnelles, vous décrivez l’action que réalise le lien ou le bouton. La pertinence d’une description spécifique d’une image dépend du contexte, mais les images informatives comme les images fonctionnelles doivent impérativement comporter une alternative textuelle.
