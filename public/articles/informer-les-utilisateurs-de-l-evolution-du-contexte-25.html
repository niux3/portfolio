Lorsque un lien s'ouvre dans un nouvel onglet ou une nouvelle fenêtre de façon inattendue, cela peut causer plusieurs problèmes aux utilisateurs :

  • Il peut confondre et désorienter les personnes ayant des difficultés à percevoir le contenu visuel ou souffrant de troubles cognitifs.
  • Il n'est pas toujours évident de savoir qu'un lien s'est ouvert dans un nouvel onglet, en particulier sur les navigateurs mobiles, ce qui peut être désorientant.
  • Le bouton retour peut ne pas fonctionner parce que tous les navigateurs ne partagent pas l'historique de navigation d'une session entre les onglets. Si l'utilisateur ne peut pas utiliser le bouton retour, il doit trouver le moyen de revenir à la page précédente.
  • Les utilisateurs moins techniques peuvent ne pas comprendre comment naviguer entre les fenêtres et les onglets.
  • En plus de cela, il encombre l'espace d'information de l'utilisateur.

Si l'ouverture d'un lien dans un nouvel onglet ou une nouvelle fenêtre améliore l'expérience de l'utilisateur et que vous utilisez target="_blank" sur un lien, il est préférable d'informer les utilisateurs dans le texte du lien que le lien s'ouvrira dans un nouvel onglet, comme le montre l'exemple 1.

Exemple 1 – Un texte de lien qui comprend un avertissement entre parenthèses.

<a href="https://www.bob-eponge.fr" target="_blank">
  Le site de  Bob l'éponge (ouverture nouvel onglet).
</a>

Si vous ne pouvez pas ajouter d'avertissements manuellement et que vous n'avez pas de logique d'automatisation côté serveur en place, vous pouvez utiliser le CSS. C'est mieux que de ne pas avoir d'avertissement, mais cette solution présente des inconvénients:

  • Sur les sites comportant plusieurs langues, vous devez gérer les traductions du texte d'avertissement en CSS et non en HTML ou dans une base de données. Ce n'est pas intuitif et c'est une source d'erreur parce qu'en général, vous ne vous attendez pas à un contenu textuel en CSS.
  • Les outils de traduction automatique risquent de ne pas traduire le pseudo-contenu.
  • Les informations ne sont pas disponibles si le CSS ne parvient pas à se charger ou si la page est présentée sans CSS.

L'exemple 2 semble être une solution élégante, mais je recommande d'utiliser le texte de l'élément directement au lieu d'un texte provenant d'un pseudo-élément.

Exemple 2 – Un avertissement ajouté par l'intermédiaire d'un pseudo-élément

<a href="https://www.bob-eponge.fr" target="_blank">
  Le site de  Bob l'éponge (ouverture nouvel onglet).
</a>

[target="_blank"]::after {
  content: " (ouverture nouvel onglet)";
}

Note : Un élément normal en HTML est un élément que vous ajoutez à une page directement dans votre document HTML ou via JavaScript. Un pseudo-élément ne fait pas partie de ton document HTML mais est ajouté au DOM via CSS.

Est-ce une bonne idée de forcer l'ouverture des liens dans de nouveaux onglets ou de nouvelles fenêtres ? Selon la personne à qui vous posez la question, vous obtiendrez des réponses très différentes. Certaines personnes favorisent l'ouverture de liens externes dans de nouvelles fenêtres parce que cela permet aux utilisateurs de rester sur votre site. Ce sur quoi les experts en utilisabilité et en accessibilité sont d'accord, c'est que vous devez l'éviter (à quelques exceptions près) et informer les utilisateurs si vous ne le faites pas.

Que les liens s'ouvrent dans de nouvelles fenêtres ou de nouveaux onglets peut être problématique pour la convivialité et l'accessibilité n'est pas une idée nouvelle. Depuis, nous avons vu apparaître beaucoup plus d'appareils portables comme les smartphones et les tablettes, où l'espace est limité et où garder une trace des onglets et des fenêtres ouvertes est encore plus difficile.

Outre les problèmes énumérés dans la section de cette recette consacrée aux problèmes, l'un des arguments les plus convaincants contre l'ouverture des liens dans un nouvel onglet est que lorsque vous appliquez la méthode target="_blank" vous enlevez à l'utilisateur la possibilité de décider comment ouvrir un lien. Il s'ouvrira toujours dans un nouvel onglet ou une nouvelle fenêtre (selon les paramètres du navigateur).

Le site (WCAG) énumère des scénarios dans lesquels le fait de forcer les liens à s'ouvrir dans un nouvel onglet est préférable pour l'expérience de l'utilisateur :

  • Si une page liée contient des informations contextuelles destinées à persister à côté du contenu principal, comme des instructions pour remplir un formulaire ou d'autres documents de référence, l'ouverture de la page dans le même onglet peut perturber un flux de travail en plusieurs étapes.
  • Si un lien ouvre des widgets dans une fenêtre contextuelle, comme un sélecteur de date basé sur un calendrier ou une connexion, vous voulez que les utilisateurs effectuent ces actions en dehors de la page principale.
  • Si un utilisateur est connecté, le fait d'accéder à une page en dehors de la zone sécurisée pourrait mettre fin à la session de l'utilisateur.

Dans les cas où l'ouverture d'un nouvel onglet par défaut aide les utilisateurs, prévoiyez un avertissement. Cela leur permet de décider s'ils veulent quitter la fenêtre actuelle, et cela les aidera à s'orienter et à naviguer.

La façon la plus fiable d'ajouter un avertissement est d'utiliser du texte et un message d'avertissement rapide, comme "S'ouvre dans un nouvel onglet", comme le montrent les exemples 1 et 2. Ces deux solutions ajoutent du contenu à l'étiquette d'origine et ne l'écrasent pas. Une mauvaise pratique courante consiste à utiliser aria-label pour cette tâche, comme le montre l'exemple 3. aria-label écrase le texte original et rend le lien pratiquement inutile parce que son nom accessible est maintenant « S'ouvre dans un nouvel onglet ».

Exemple 3 – Mauvaise pratique : aria-label écrase l'étiquette originale

<a href="https://www.bob-eponge.fr" target="_blank"  aria-label="ouverture nouvel onglet">
  Le site de  Bob l'éponge.
</a>

Si vous définissez un lien pour qu'il s'ouvre dans un onglet, basez votre décision sur ce qui est le mieux pour vos utilisateurs. Le référencement ou les indicateurs clés de performance (ICP) ne devraient pas entrer en ligne de compte. Si possible, étayez vos choix par des tests de convivialité ou des enquêtes contextuelles.
