Fournir des styles de focalisation

Les pages Web sont accessibles au clavier par défaut parce que les éléments interactifs natifs sont livrés avec le style et la fonctionnalité dont vous avez besoin pour les utiliser. Ils sont focalisables et indiquent visuellement leur état de focalisation. Lorsque vous ajoutez du CSS ou surtout du JavaScript, vous devez veiller à conserver cette accessibilité de base, voire à l'améliorer, et à ce que vos solutions personnalisées soient également accessibles.

Lorsqu'on accède à un site Web à l'aide d'un clavier, d'un dispositif de commutation, d'un lecteur d'écran ou d'une technologie d'assistance similaire, on peut utiliser la touche Tab (ou une commande sur un autre dispositif physique qui correspond à la touche) pour passer d'un élément interactif à l'autre.

Si, en tant que développeur, vous ne mettez pas visuellement en évidence l'élément actuellement actif à l'aide de CSS, les utilisateurs ne peuvent pas s'orienter et naviguer.

La solution est d'utiliser les pseudoclasses pour styliser les éléments interactifs dans leur état de mise au point ou de mise au point visible, comme le montrent les exemples 1, 2, 3 et 4.

Exemple 1 – Styliser tous les éléments dans leur état focus-visible

:focus-visible {
  outline: 0.25em solid;
  outline-offset: 0.25em;
}

Exemple 2 – Styliser tous les éléments dans leur état de focalisation

:focus {
  outline: 0.25em solid;
  outline-offset: 0.25em;
}

Exemple 3 – Affichage d'une ombre sur les éléments vidéo et audio si un élément contenu est actuellement mis en avant.

:is(video, audio):focus-within {
  box-shadow: 0 0 10px 3px rgb(0 0 0 / 0.2);
}

Exemple 4 – Fournir des styles de mise au point personnalisés pour les utilisateurs n'utilisant pas le clavier

button:focus-visible {
  outline: 0.25em dashed black;
}

button:focus:not(:focus-visible) {
  outline: none;
  box-shadow: 1px 1px 5px rgba(1, 1, 0, .7);
}

Remarque : Le premier sélecteur (button:focus-visible) et sa règle CSS définissent le style de mise au point pour les utilisateurs naviguant au clavier. Le second sélecteur (button:focus:not(:focus-visible)) s'applique aux utilisateurs interagissant à la souris.

Le style de l'élément sur lequel on se concentre actuellement est l'une des choses les plus importantes que vous puissiez faire pour l'accessibilité du clavier. Si vous supprimez les styles de mise au point par défaut fournis par tous les navigateurs, vous compliquez la vie de nombreuses personnes. Supprimer les indicateurs de mise au point pour les utilisateurs de clavier revient à cacher le curseur pour les utilisateurs de souris. Malheureusement, vous trouverez le code de l'exemple 5 sur de nombreux sites Web. C'est parce que, dans le passé, l'ajout de styles de mise au point avait des effets secondaires indésirables pour les utilisateurs de souris, comme l'affichage d'un contour que les parties prenantes considèrent souvent comme esthétiquement déplaisant ou dérangeant (ce sont rarement les utilisateurs qui se plaignent). Heureusement, ce problème appartient au passé. Aujourd'hui, vous pouvez contrôler précisément quand et à qui les styles de mise au point sont visibles, car les navigateurs utilisent différentes heuristiques pour les afficher.

Exemple 5 – Mauvaise pratique : Pas de contour par défaut sur les éléments interactifs.

:focus {
  outline: none !important;
}

La pseudoclasse :focus

La pseudoclasse :focus applique lorsque l'utilisateur met l'accent sur un élément à l'aide du clavier, de la souris ou de toute autre forme de saisie. En utilisant le code de l'exemple 2, le contour personnalisé de 0,25em s'affiche lorsque l'utilisateur trouve le bouton à l'aide du clavier et lorsqu'il clique ou tape dessus. Auparavant, :focus figurait dans la feuille de style de l'agent utilisateur de tous les navigateurs jusqu'à ce qu'ils adoptent :focus-visible comme valeur par défaut.

La pseudoclasse :focus-visible

À partir de avec Chrome 90, Firefox 87 et Safari 15.4, la pseudoclasse :focus correspond toujours aux éléments focalisables, mais les agents utilisateurs n'indiquent visiblement la focalisation que parfois. Au lieu de cela, ils utilisent par défaut :focus-visible, qui utilise une variété d'heuristiques pour indiquer le focus uniquement lorsque c'est le plus utile pour les utilisateurs. Il s'agit d'un changement crucial, car vous pouvez désormais donner un style aux indicateurs de mise au point sans trop vous soucier du moment où ils s'affichent. Si vous voulez modifier les styles par défaut, vous pouvez utiliser la pseudoclasse :focus-visible, comme le montre l'exemple 1.

Il existe des différences mineures entre les moteurs de navigateur, mais les styles de mise au point ne doivent s'afficher que dans certaines circonstances, comme par exemple :

  • Si l'utilisateur interagit avec la page à l'aide d'un clavier ou d'un autre dispositif non pointu.
  • Si l'élément prend en charge la saisie au clavier, comme un élément de saisie ou une zone de texte.
  • Si vous déplacez le focus avec JavaScript, l'élément précédemment ciblé et le nouvel élément focalisé afficheront tous deux l'indication de focus..
  • Si l'utilisateur a exprimé une préférence dans les paramètres de son navigateur (par exemple, dans Chrome, sous Préférences, Accessibilité, « Afficher un surlignage rapide sur l'objet focalisé »).

Bien qu'il n'y ait pas de styles par défaut pour la pseudoclasse :focus, vous pouvez toujours l'utiliser si vous souhaitez afficher l'indicateur quelle que soit la méthode de saisie. Vous pouvez même combiner les pseudoclasses, comme le montre l'exemple 4. La première règle s'applique lorsque l'utilisateur interagit avec le bouton à l'aide d'un dispositif non pointant, et la seconde ne s'applique que lorsque l'utilisateur utilise un dispositif pointant.

La pseudoclasse :focus-within

La pseudoclasse :focus-within applique à tout élément qui correspond à la pseudoclasse :focus. Plus important encore, elle s'applique à un élément dont les descendants répondent aux conditions de correspondance avec :focus. Cela signifie que vous pouvez l'utiliser pour sélectionner un élément dont les enfants sont actuellement focalisés, comme illustré dans l'exemple 3.

Styles de mise au point par défaut

Les navigateurs sont livrés avec des styles par défaut pour les éléments focalisables. Ces styles sont souvent appelés focus-ring car ils utilisent tous la propriété outline, qui ajoute un contour autour du rectangle de l'élément. Les styles par défaut sont mieux que rien, mais ils ne sont généralement pas assez visibles, ils manquent de contraste de couleur et leur style n'est pas cohérent sur les navigateurs. C'est pourquoi je te recommande de créer vos propres règles.

La règle de l'exemple 1 utilise un outline-width plus épais et un outline-offset supplémentaire pour ajouter de l'espace entre le contenu et le contour. Elle s'assure également que répond aux exigences des WCAG en matière de contraste des couleurs, qui stipulent que l'indicateur visuel de focalisation doit présenter un contraste suffisant par rapport aux couleurs adjacentes.

En outre, les propriétés outline fonctionnent bien pour le style de mise au point parce qu'elles sont bien prises en charge. Elles sont clairement visibles et n'interfèrent pas avec la mise en page car, contrairement à border, elles n'ajustent pas la taille de la boîte. Mais vous n'êtes pas limité à ces propriétés. Vous pouvez aussi utiliser des propriétés comme background-color ou box-shadow. Cependant, il est important de savoir que ces propriétés peuvent ne pas être visibles en mode couleur forcée.

Une excellente solution consiste à combiner d'autres propriétés avec un contour transparent, comme le montre l'exemple 6. Les modes de couleur forcée peuvent ne pas rendre les couleurs d'arrière-plan et les ombres de boîte, mais ils peuvent interpréter et afficher les contours transparents.

Exemple 6 – Un contour invisible pour tous les éléments focalisés

:focus-visible {
  --black: #0b0c0c;
  --yellow: #fd0;

  color: var(--black);
  background-color: var(--yellow);
  box-shadow: 0 -0.125em var(--yellow), 0 0.25em var(--black);
  outline: 0.25em solid transparent; 
}

Lorsque vous concevez les styles de mise au point des éléments interactifs de votre projet, concentrez-vous sur l'expérience de l'utilisateur. Ils n'ont pas besoin d'être beaux, mais ils doivent bien faire leur travail .
